Overview Page

In this overview page you can find keyterms as well some of the syntax that Git uses. It will all be from a linux console.

These are the list of keyterms used when working with Git

  • commit - A commit is a confirmd changes to a repository, such as by modifying files, adding, removing them, etc.
  • remote - A remote is typically a repository located in a website that uses Git, such as GitHub, GitLab, BitBucket, SourceForge, AWS CodeCommit, and a list of others.
  • local - Local repository is a place inside the operating system.
  • repository - This is a place where your files are stored and kept track of such as modifying, adding, combining, and more.
  • pushing - This is when someone uploads their files from local to remote repository.

These are a list of the commands in linux to use git

  • git init - This command is typically used once at the beginning of creating a local repository.
  • git clone remoteRepository - This is used for getting a complete copy of the remote repository which could be located in websitse such as GitHub, etc.
  • git log - This shows a list of the commits that have been done.
  • git push remoteName branchName This command pushes the things in the local repository to the remote one.
  • git add - This command stages all the changes in a repository from the files.
